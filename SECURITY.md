# Security Policy

This is a static, frontend-only project. All rendering logic executes client-side inside the user's web browser, and there are no servers or database dependencies. However, we take security seriously and want to ensure a safe environment for everyone.

## Reporting a Vulnerability

If you discover a security issue or vulnerability (e.g. dynamic injection risks in the tooltip HTML generation, insecure scripts configuration, or server hosting configurations):
1. Please **do not** open a public issue.
2. Email your findings directly to the project maintainer at `security@jayshilsanalytics.com`.
3. Provide a clear description of the vulnerability, step-by-step instructions to reproduce it, and any proof-of-concept scripts or visual snapshots.

## Disclosure Timeline

We will review your submission and aim to respond within 48 hours. If the report is validated, we will publish a fix in the repository within 7 business days, after which we will publicly announce the resolution.

Thank you for helping keep this project secure!
